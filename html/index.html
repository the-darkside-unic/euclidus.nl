Onze hoofdvraag was: Hoe kan je een Raspberry pi opstellen voor een
server die een site geeft bij het benaderen en die veilig is.

Om die vraag te kunnen beantwoorden hadden we ook een paar theorie en praktijk vragen.

Theorie vragen


Theorie vraag 1: Wie of wat is een webhost?

Webhosting is een dienst waarbij een bedrijf of particulier informatie, afbeeldingen of dergelijken kan opslaan waardoor die van buitenaf benaderbaar is op een website.

Soorten webhosting

Gratis hosting: In dit geval is het gratis, maar vaak heeft de klant dan maar beperkte mogelijkheden en of staan er advertenties op de website van de klant.
Hidden webhosting: Hierbij staat de site op het dark web en daardoor alleen benaderbaar met de Tor-browser.
Shared hosting: Bij shared hosting worden er op een server meerdere websites geplaatst. Een nadeel hiervan is dat bij problemen al die sites last ondervinden. Bij bijvoorbeeld een probleem met een van die sites die ervoor zorgt dat de server crashed. Dan hebben alle gebruikers er last van.
Reseller hosting: Hierbij ben jij degene die webhosting host. Je moet dan zelf beschikken over grootte bandbreedte en een hoge schrijfruimte.
Virtual Private Server (VPS) hosting: Hierbij draaien er meer virtuele servers op een server waardoor de verschillenden gebruikers root rechten hebben op hun persoonlijke virtuele server. Als een van de virtuele servers dan crashed dan hebben andere gebruikers daar geen last van.
Dedicated hosting: In dit geval krijgt de klant echt zelf de server en de rechten en is het niet een virtuele server, maar moet de klant zich wel houden aan een datalimiet en schrijfruitme.
Managed hosting: Hierbij krijgt de klant een eigen server maar het onderhoud en opzetten hiervan wordt uit handen gegeven.
Co-located hosting: In dit geval plaats de gebruiker zelf de server bij een hoster
Cloud hosting: Hierbij worden meerdere servers aan elkaar gekoppeld voor het hosten en hierdoor is een site ook makkelijk schaalbaar waardoor het vermogen van een website dus automatisch groter kan worden als er meer dan gemiddeld aantal bezoekers komen.

Theorie vraag 2 en 3: Hoe staat een site opgeslagen op een server? en hoe communiceert de server met de gebruiker en andersom?

Het internet is een web van computers. Via dit web is je computer verbonden aan de server van de website die je bezoekt, via het netwerk wordt de benodigde informatie gestuurd naar de computer van de gebruiker. De verbonden computers zijn ongeveer in 2 functies te verdelen. Je hebt servers en client computers. Client computers zijn bijvoorbeeld je telefoon, tablet of pc. Servers zijn bepaalde computers met bepaalde functies. Een server kan bijvoorbeeld een webpagina weergeven. Deze pagina staat dan op de harde schijf van de server. Wanneer een client-computer (een gebruiker) deze webpagina wil bekijken moet hij eerst op zoek naar de goede server en haalt daar dan de gegevens van de server als de server dat toe staat. .



Praktijkvragen



Praktijk vraag 1: Hoe stel je de router goed in?

Praktijk verslag Hoe stel je de router goed in? Zet de router aan doe de LAN-kabel erin doe de andere kant in je server (de raspberry pi). Maar dat is nog niet alles ook zal je moeten port forwarden.
Ook moet je ervoor zorgen dat de server buiten de interne firewall van je thuis netwerk staat. Anders kunnen hackers zo, vanuit je server je LAN-netwerk binnendringen. Door je server buiten de firewall te plaatsen kan je aanvallen ook snel herkennen en afslaan. Je moet er natuurlijk wel voor zorgen dat er niet belangrijke dingen op de server staan, want er is natuurlijk geen firewall om mensen die die informatie willen hebben tegen te houden.


Port Forwarden

Port forwarden Port forwarding, ook wel port mapping genoemd, is een methode voor het doorsturen van een netwerkpoort van het ene netwerkknooppunt naar het andere. Met deze techniek kan een externe gebruiker van buitenaf een poort op een prive IP-adres (binnen een LAN) bereiken met behulp van een NAT-compatibele router. .

Praktijk vraag 2: Hoe koppel je een hostnaam

Hoe koppel je een hostnaam? Elke computer die op het internet aangesloten is, is benaderbaar door een 12 cijferig IP-adres. Maar omdat dat niet zo makkelijk te onthouden is kan je jouw IP-adres een naam geven. Dit heet een hostnaam.
Om te zorgen dat deze naam ook echt verwijst naar het goede IP-adres, word het opgeslagen op een zogenoemde DNS. DNS zijn grote databases waar opgeslagen staat welke naam bij welk IP hoort. Om een hostnaam te koppelen ga je naar een zogenaamde DNS-provider. Soms kosten deze geld, maar dat hoeft niet. Wij gebruiken Trans ip.

SSH

Via SSH (Secure Shell) kan je aanloggen op een systeem vanuit een andere computer. Dit is erg handig als je server op een externe locatie is. Je kan bijvoorbeeld met SSH aanloggen op de raspberry pi vanuit school terwijl de pi thuis staat. .

Praktijk vraag 3: Hoe zorg je voor een werkende server?

Hoe zorg je voor een werkende server?

  1. Flash het sd-kaartje van de pi. Je kan hier een programaatje genaamd Etcher. Dit programma maakt het een stuk makkelijker.
  2. Installeer een server op je pi hier gebruiken wij Apache Server voor. Dit programma zorgt ervoor dat als er een verzoek om een resource binnenkomt, dat er dan een antwoord op dat verzoek verstuurd. Bijvoorbeeld een bestand.
  3. Om het verkeer op de site veilig te maken heb je een SSL-certificaat nodig. Dit certificaat zorgt ervoor dat de connectie tussen de gebruiker en de site versleuteld is.
  4. Stel je router zo in dat de server berijkbaar is van buiten, door alleen bepaalde poorten door te laten zorg je voor toegenomen veiligheid op de server. Ook is het handig als je de server buiten je interne firewall plaatst(mits het niet al te belangrijke informatie bevat). Er moet ook zo min mogelijk communicatie plaats vinden tussen de server en jouw thuis netwerk, want dan is het een stuk makkelijker voor hackers om binnen te dringen in je LAN-netwerk.

In github werken maakt alles een stuk makkelijker, vooral als je met een team van mensen werkt. Maar hoe zorg je ervoor dat alles wat je op github pushed ook echt op de site terecht komt?
  1. Maak een mapje aan op de server en noem deze git
  2. Maak in github een project voor je server aan, maak hierin een mapje aan waar je alle code in gaat zetten.
  3. Om te zorgen dat je website altijd up to date blijft moet je het project github clonen naar je server. Dit doe je door het git mapje te openen in een terminal op je server, en dit commando in te voeren git clone "https://github.com/jouw-team/jouw-project.git"
