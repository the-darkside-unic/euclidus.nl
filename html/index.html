Hoofdvraag:

Onze hoofdvraag was: Hoe kan je een Raspberry pi opstellen voor een
server die een site geeft bij het benaderen en die veilig is.

Om die vraag te kunnen beantwoorden hadden we ook een paar theorie en praktijk vragen.

Theorie vragen:

   1. Wie of wat is een webhost?
   2. Hoe staat een site opgeslagen op een server (raspberry Pi)?
   3. Hoe communiceert de site met de gebruiker en andersom?

Praktijkvragen:

   1. Hoe stel je de router goed in?
   2. Hoe koppel je een domeinnaam?
   3. Hoe zorg je voor een werkende server?

Theorie vraag 1: Wie of wat is een webhost?

Webhosting is een dienst om informatie op te slaan die dan toegankelijk is vanaf buiten. Wat het eigenlijk is is dat je de bestanden met de regels code opslaat op die computer. En dat die dan via een URL (Bijv. google.nl) te bezoeken is.

Theorie vraag 2: Hoe staat een site opgeslagen op een server (raspberry Pi)?

Op de server staat de site (dus als bestanden met regels code) opgeslagen op een harde schijf. (opslag). De site bestaat uit een front en een backend. De front-end is wat de gebruiker te zien krijgt, denk aan animaties lay-out plaatjes en tekst voor de gebruiker. De backend is alle berekeningen en versturen en ontvangen van informatie en dergelijken. Alles wat nodig is voor de site, maar wat de gebruiker niet ziet behoort dus tot de backend.

Theorie vraag 3: Hoe communiceert de site met de gebruiker en andersom?

De site en gebruiker communiceren over het World Wide Web denk aan WWW.xxxxx.com.
Dit doen ze door het verzenden van pakketjes met informatie naar elkaar. Bijvoorbeeld wat een gebruiker heeft ingevoerd. Als er in de URL http:// staat dan zijn die pakketjes in te zien als ze worden onderschept. Staat er https:// (met dus een s erbij) dan worden de pakketjes versleuteld. En zijn ze dus niet te lezen als ze onderschept worden. Let dus altijd op dat als je prive zaken wilt versturen op een site dat er aan het begin van de URL https:// staat.

Praktijk vraag 1: Hoe stel je de router goed in?

De router moet je zo instellen dat de server (in dit geval raspberry Pi) voor de verdedigingslinie zit. En dus te bezoeken vanuit buitenaf. Alles wat achter de verdedigingslinie zit is alleen te zien van binnen het netwerk. Als de router dus niet goed is ingesteld kan je op je eigen netwerk de site bekijken, maar niet vanuit buitenaf.

Praktijk vraag 2: Hoe koppel je een domein naam?

Dit doe je door middel van een DNS-server. Een DNS-server is eigenlijk een groot telefoonboek voor sites. Want eigenlijk heeft eigen site zijn eigen IP-adres (in ons voorbeeld het telefoonnummer), maar omdat het zo lastig is die te onthouden heb je namen voor sites. Dus als je www.google.com intypt dan zoekt die DNS-server naar het telefoon nummer dat bij die naam hoort en dan stuur die DNS server je naar het juiste telefoon nummer. (In het echt is het dus een IP-adres i.p.v. een telefoonnummer, maar dat was voor het voorbeeld.

Praktijk vraag 3: Hoe zorg je voor een werkende server?

Eerst moet je een besturingssysteem downloaden en installeren op de server. (Een besturingssysteem is zoiets als Windows of Mac OS X) Daarna moet je ervoor zorgen dat als je die server benaderd die de juiste regels code doorstuurt hier heb je programmatjes voor. Die moet je ook installeren.

Die regels code staan opgeslagen in bestanden. Deze bestanden hebben wij in ons geval op een andere site opgeslagen (Git Hub) dit is een site voor programmeurs en software bouwers. Hierdoor konden we vanaf onze laptops de bestanden daarop zetten. En als de Server aanstaat haalt die de bestanden van die site en is de site altijd up-to-date.
